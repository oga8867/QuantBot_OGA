## 1. 한눈에 보는 봇 개요

### 🎯 이 봇이 하는 일

1. **종목 분석**: 한국/미국 주식의 가격·재무·뉴스를 동시에 분석
2. **자동 매매**: 매수/매도 신호 발생 시 자동으로 주문 실행
3. **자동 청산**: 손절/익절/트레일링 스탑이 모두 자동
4. **시장 위기 회피**: 서킷브레이커/사이드카/VI 자동 감지로 매매 차단
5. **실시간 모니터링**: 대시보드 + 디스코드 봇으로 어디서든 확인

### 📊 한국 + 미국 동시 운영

| 시장 | 데이터 소스 | 매매 브로커 | 실시간 |
|------|----------|----------|------|
| 🇰🇷 한국 (KOSPI/KOSDAQ) | KIS API + pykrx | 한국투자증권 (KIS) | ~1초 (KIS) |
| 🇺🇸 미국 (NYSE/NASDAQ) | yfinance | Alpaca (선택) | ~15분 (무료) |

### 🛡️ 안전을 최우선으로 설계

- 모의투자 1억원 무료 (한국투자증권 제공)
- 실거래 전 2-4주 모의 검증 권장
- 7중 안전장치 (일일 손실 한도, 킬스위치, CB 감지 등)
- 모든 거래 결정 근거 자동 저장 + 클릭으로 상세 확인

## 3. API 키 입력 (5분)

### 🔑 두 가지 입력 방법 (편한 것 선택)

#### 방법 A: API_KEYS.txt 메모장 편집 ⭐ (가장 쉬움)

폴더에서 `API_KEYS.txt` 더블클릭 → 메모장 자동 열림 → 따옴표 안에 키 붙여넣기:

```
KIS_APP_KEY = 'PSlawusu...'        ← 따옴표 안에만
KIS_APP_SECRET = 'fOmb9rV09...'
KIS_ACCOUNT = '12345678-01'
KIS_PAPER = 'true'                  ← 처음엔 반드시 true!
```

저장 → 봇 재시작 → 자동 로드.

#### 방법 B: 대시보드 설정 탭
대시보드 → "설정" 탭 → 각 항목 입력 → "저장" 버튼

| API | 발급처 | 용도 |

| **KIS App Key + Secret** | apiportal.koreainvestment.com | 한국 매매 + 실시간 시세 |
| **KIS 계좌번호** | 한국투자 앱 → 계좌정보 | (위와 동일) |
| Alpaca | alpaca.markets | 미국 매매 (선택) |
| FRED | fred.stlouisfed.org | 거시경제 데이터 (선택) |
| DART | opendart.fss.or.kr | 한국 공시 (선택) |
| Telegram Bot | @BotFather | 알림 (선택) |
| Discord Webhook | 디스코드 채널 설정 | 알림 (선택) |

### ⚠️ 모의투자 먼저!

처음에는 반드시 `KIS_PAPER = 'true'`로 설정.
- 한국투자 앱 → "모의투자" 메뉴 → 신청 → **즉시 1억원 가상 자본 지급**

## 5. 핵심 기능 12가지

### 🧠 1) 멀티 알파 앙상블 엔진
3개 분석 모듈 동시 사용 → 가중 평균으로 최종 신호:

| 모듈 | 가중치 | 분석 내용 |
|------|------|---------|
| 기술적 분석 | 45% | RSI, MACD, 볼린저밴드, SMA, EMA, OBV, ATR |
| 팩터 분석 | 35% | Value, Quality, Momentum |
| 뉴스 감성 | 20% | Google News RSS 키워드 분석 |

**수식**: `Σ(score_i × weight_i × confidence_i) / Σ(weight_i × confidence_i)`

비활성화 시 자동으로 가중치 재정규화 → 합계 항상 1.0.

### 🎯 2) 적응형 임계값 (VIX 기반)
시장 변동성에 따라 매수 기준 자동 조절:

| VIX | 시장 체제 | 매수 임계값 | 포지션 크기 |
|-----|---------|---------|---------|
| < 15 | 안정 | 0.14 (낮음, 적극적) | 1.0× |
| 15~20 | 정상 | 0.20 (표준) | 1.0× |
| 20~30 | 불안 | 0.30 (보수적) | 0.7× |
| > 30 | 위기 | 0.50 (매우 보수) | 0.3× |

추가로 시장 추세(Bull/Bear/Sideways)에 따라 보정:
- Bear장: 매수 임계값 ×1.2 (더 엄격), 포지션 ×0.7
- **VIX > 30 + Bear장 → 매수 자체 차단**

### 🛡️ 3) 자동 청산 시스템 (ExitManager) ⭐ 핵심 기능
진입 후 5가지 트리거로 자동 매도:

| 트리거 | 조건 | 매도 비중 |
|--------|------|---------|
| 🔴 손절 (Stop Loss) | 현재가 ≤ 진입가 - ATR×2 | 100% |
| 🟢 1차 익절 | 현재가 ≥ 진입가 + ATR×4 | 50% + 손절선 본전 상향 |
| 🟢 2차 익절 | 현재가 ≥ 진입가 + ATR×8 | 잔여 100% |
| 📉 트레일링 스탑 | 1차 익절 후, 최고가 - ATR×3 도달 | 잔여 100% |
| 📊 앙상블 SELL | 모듈 종합 점수 < -0.2 | 100% |

**학술 근거**: Le Beau Chandelier Exit (1992) — ATR×3 트레일링이 드로우다운 22% 감소.

### 📊 4) 시장 정지 자동 감지 (실거래 필수)

| 메커니즘 | 발동 조건 | 봇 동작 |
|--------|---------|--------|
| 서킷브레이커 1단계 | KOSPI -8% / S&P -7% | 모든 매매 차단 20분 |
| 서킷브레이커 2단계 | KOSPI -15% / S&P -13% | 추가 20분 차단 |
| 서킷브레이커 3단계 | KOSPI -20% / S&P -20% | 당일 매매 종료 |
| 사이드카 | KOSPI -5% | 5분 차단 |
| VI (개별 종목) | 종목별 급변 | 해당 종목만 보류 |

5단계 보호 메커니즘 + 디스코드/텔레그램 즉시 알림.

### 💰 5) Half-Kelly 포지션 사이징
Kelly Criterion 변형으로 포지션 크기 자동 계산:
```
shares = (capital × kelly_fraction × confidence) / (price × ATR_multiplier)
kelly_fraction = 0.5 (Half-Kelly, 변동성↓)
```

학술 근거: Half-Kelly가 75% 성장률 + 변동성 대폭 감소 (Thorp 1969).

### 🔄 6) 종목 자동 발굴
워치리스트 외 유망 종목 자동 탐색:
- 섹터 유니버스(~100종목) + 시장 거래량 상위
- 6가지 신호: 거래량 급증, RSI 극단, 급등/급락, 볼린저 돌파, MACD 크로스, 52주 신고/신저
- N번째 분석 사이클마다 1회 실행 (기본 4회 = 1시간마다)
- 연속 HOLD 종목 자동 제거 (회전)

### 📈 7) 실시간 시세 (KIS API)
한국 주식: **~1초 지연** (vs yfinance 15-20분 지연)
- 보유 포지션 가격 1분마다 갱신
- 토큰 자동 캐싱 (24시간 재사용 → SMS 빈도 24시간 1회로 감소)
- KIS 자격증명 없으면 자동으로 yfinance 폴백

### 💬 8) Discord 양방향 봇
디스코드에서 봇 직접 제어:
- `/상태` - 봇 상태 + 자산 현황
- `/포지션` - 현재 보유 종목
- `/거래내역` - 최근 거래 + 오늘 손익
- `/시작` /`/중지` - 봇 시작/중지
- `/분석 005930` - 즉석 종목 분석
- `/보고서` - 일일 보고서 생성
- `/도움말` - 명령어 목록

### 📋 9) 거래 결정 상세 모달
거래 이력의 "전략" 컬럼 클릭 → 풍부한 결정 근거 표시:
- 앙상블 점수 + 신뢰도 + 모듈별 기여도 막대그래프
- 적응형 임계값 (VIX, 시장 체제, 포지션 배수)
- 진입 시점 지표 (RSI, MACD, SMA, ATR)
- 매도 시: 손익 분석 + ExitManager 상태 + 진입가→청산가

### 🔍 10) 거래 이력 필터링 + 손익 분석
- 종목 검색 (코드/이름 부분 일치)
- 매수/매도 필터
- 손익 필터 (수익만/손실만)
- 상단 카드: 오늘 손익 + 전체 누적 손익 (승률, 손익비)

### 📊 11) Buy-and-Hold 벤치마크
일일 보고서에 자동 비교:
- 봇 수익률 vs 벤치마크 (KODEX 200 / SPY / ACWI 자동 선택)
- 알파 (초과 수익률) + 정보비율
- 봇/벤치 샤프비, MDD 비교
- 등급 A (의미 있게 초과) / B (비슷) / C (미달 → 전략 재검토)

### 🧪 12) 회귀 테스트 42개

```bash

```

## 6. 매매 전략 작동 원리

### 분석 사이클 흐름 (기본 1시간마다)

```
[1] 워치리스트 결정
    = 사용자 관심종목 + 자동 발굴 + 보유 종목

[2] 시장 정지 감지 ⭐
    KOSPI/S&P 변동률 체크 → CB 발동 시 사이클 스킵

[3] 적응형 임계값 갱신
    VIX 조회 → 매수 임계값 / 포지션 크기 결정

[4] 각 종목 분석
    ┌─ 1. 가격 데이터 수집 (yfinance + KIS)
    ├─ 2. 기술적 분석 (RSI/MACD/BB/ATR/...)
    ├─ 3. 팩터 분석 (Value/Quality/Momentum)
    ├─ 4. 뉴스 감성 분석 (Google News RSS)
    ├─ 5. 앙상블 결합 → BUY/SELL/HOLD
    └─ 6. 각 신호를 DB에 저장

[5] 보유 종목: ExitManager 평가 (우선)
    손절/익절/트레일링 → 발동 시 즉시 매도

[6] 매수 신호 처리
    ┌─ 적응형 임계값 통과 확인
    ├─ 매도 쿨다운 체크 (1시간)
    ├─ 포지션 유형 분류 (단타/스윙/장기)
    ├─ 비활성 유형이면 매수 차단
    ├─ Kelly 사이징 + 적응형 배수
    ├─ SafetyGuard 검사 (한도, 킬스위치)
    └─ 매수 실행 + ExitManager 진입 등록

[7] 매도 신호 처리
    ┌─ 시장 정지 시 차단
    ├─ SafetyGuard 검사
    └─ 매도 실행 + 쿨다운 등록

[8] DB 동기화 + 알림 전송
```

### 포지션 유형 자동 분류

매수 시 신호 특성에 따라 3가지로 분류:

| 유형 | 분류 기준 | 손절 | RR | 보유 기간 |
|------|---------|----|----|---------|
| 🔵 단타 (Short) | 기술적 신호 지배 | ATR×1.5 | 1.5 | 1~3일 |
| 🟡 스윙 (Swing) | 혼합 신호 (기본) | ATR×2.0 | 2.0 | 1~4주 |
| 🟢 장기 (Long) | 펀더멘탈 지배 | ATR×3.0 | 3.0 | 1개월+ |

→ 사용자가 설정에서 각 유형을 ON/OFF 가능. OFF 시 다음 우선순위로 fallback (스윙 → 장기 → 단타).

## 7. 자동 청산 시스템 (ExitManager)

### 5가지 매도 트리거 (우선순위 순)

```
매 분석 사이클마다 보유 포지션 평가
        ↓
   ① 트레일링 스탑 갱신 (1차 익절 후)
        ↓
   ② 손절/트레일링 발동 체크
        ├─ 분할 익절 전 → 🔴 STOP_LOSS
        └─ 분할 익절 후 → 📉 TRAILING_STOP
        ↓
   ③ 1차 익절 체크 (분할 50%)
        → 🟢 TAKE_PROFIT_1
        → 손절선을 본전(진입가)으로 상향
        ↓
   ④ 2차 익절 체크 (전량)
        → 🟢 TAKE_PROFIT_2
        ↓
   ⑤ 시간 기반 청산 (옵션, 기본 OFF)
        → ⏰ TIME_STOP
```

### 핵심: 1차 익절 후 무위험 거래 모드

```
Day 1: 진입 ₩100,000
       손절: ₩96,000 (ATR×2 = ₩4,000)
       1차 목표: ₩108,000

Day 5: ₩108,000 도달 → 1차 익절 50% 매도
       손절선 → ₩100,000 (본전 상향) ★ 무위험 모드 시작
       트레일링 스탑 활성화

Day 7: ₩120,000 (최고가) → 트레일링 ₩114,000

Day 10: ₩116,000 (조정) → 트레일링 그대로

Day 12: ₩113,000 → 트레일링 ₩114,000 도달 → 잔여 매도
        총 수익: 50% × 8% + 50% × 13% = ~10.5% (수수료 후)
```

### 시각화: 진행률 게이지

대시보드 포지션 카드 아래 색상 막대:
- 🟢 초록: 진입가 → 1차 목표가 진행률
- 🔴 빨강: 진입가 → 손절가 진행률
- "1차 익절완료" 배지 (분할 매도 후)

## 8. 안전장치 7중 보호

### 1) SafetyGuard 모듈
모든 주문이 이 모듈을 통과해야 실행:

| 항목 | 기본값 | 효과 |
|------|------|------|
| 일일 손실 한도 | -3% | 도달 시 모든 매매 자동 중단 |
| 일일 거래 횟수 | 50건 | 초과 시 매매 차단 |
| 연속 손실 한도 | 5건 | 도달 시 킬스위치 자동 발동 |
| 종목당 최대 비중 | 20% | 한 종목 집중 방지 |
| 최대 동시 보유 | 10종목 | 분산 강제 |
| 주문 최소/최대 금액 | 자본 0.1% / 20% | 너무 작거나 큰 주문 방지 |
| 주문 지연 시간 | 3초 (실거래) | 잘못된 주문 취소 기회 |

### 2) 시장 정지 감지
서킷브레이커/사이드카/VI 자동 차단 (위 [5번 4) 참조]).

### 3) 매도 쿨다운
매도 직후 1시간 동안 같은 종목 재매수 차단:
- 톱니질(매수↔매도 반복) 방지
- 추세 전환 확신할 시간 확보

### 4) 종목 자동 발굴 한도
- 시장당 최대 발굴 종목: 10개
- 통합 워치리스트 최대: 35개
- 자동 회전 (연속 HOLD 4회 시 제거)

### 5) 환율 안전 검증
미국 주식 매매 시 USD/KRW 환율 자동 적용:
- KRW 잔고에서 환산 차감
- 환율 비정상 시 (500 미만 또는 2000 초과) 거래 차단

### 6) ATR 0 방지
가격 데이터 부족 시 ATR이 0이 되어 손절선이 무의미해짐:
- ATR 계산 실패 시 자동으로 `price × 2%` 추정값 사용
- 손절선 항상 유효

### 7) 토큰 만료 자동 갱신
KIS 토큰 만료 1시간 전 자동 재발급:
- 거래 중 인증 실패 방지
- 만료 직전 토큰으로 주문 시도 막음

### 🚨 킬 스위치 (Manual)
대시보드 → "킬 스위치" 버튼:
- 즉시 모든 신규 주문 차단
- 위급 상황 시 1초 만에 봇 정지

## 9. 전략 ON/OFF 설정

### 📅 포지션 유형 토글
"단타가 빈번해서 손해 본다" → 단타 OFF, 스윙/장기만 사용

```
[설정] → 🎯 전략 ON/OFF
  ☑ 단타 (Short)   1~3일
  ☑ 스윙 (Swing)   1~4주  ⭐ 기본
  ☑ 장기 (Long)    1개월+
```

OFF 시 동작:
- 해당 유형으로 분류된 신호 → 다음 우선순위로 fallback
- 모두 OFF → 매수 자체 차단

### 🧠 분석 모듈 토글
"뉴스가 부정확하다" → 뉴스 감성 OFF, 기술+팩터만 사용

```
☑ 기술적 분석 (45%)
☑ 팩터 분석 (35%)
☑ 뉴스 감성 (20%)
```

OFF 시 가중치 자동 재정규화. 예) 뉴스 OFF → 기술 0.563 + 팩터 0.438.

### 🔒 엄격 화이트리스트 모드
"내가 지정한 종목만 매매" → 엄격 모드 ON

```
🔒 엄격 화이트리스트 모드 ☑

[ON 효과]
- 워치리스트에 명시한 종목만 매수 가능
- 자동 발굴 종목 매수 차단
- 워치리스트 비어있어도 기본값 fallback 안 함
- 보유 종목 매도/손절/익절은 정상
```

→ "내가 명시한 종목 외엔 절대 매수 안 함" 보장.

|------|------|------|

| 봇 MDD | 최대 낙폭 | < 15% |
| 상관계수 | 봇 vs 벤치 | 낮을수록 분산 효과 ↑ |

벤치마크 자동 선택:
- 한국 비중 ↑ → KODEX 200 (069500.KS)
- 미국 비중 ↑ → SPY
- 혼합 → ACWI (MSCI All-World)

### ⚠️ 백테스트의 진실

**백테스트는 미래 성과를 보장하지 않습니다.**
- SPIVA: S&P 500을 5년 이긴 펀드는 25% 미만
- 백테스트 통과 ≠ 실거래 성공 (충분조건 아님)
- 백테스트 실패 = 실거래 실패 거의 확실 (필요조건)

**그래서 모의투자 2-4주 검증이 필수**입니다.

## 14. 트러블슈팅

### 봇 시작 시 에러

**"Python not found"**
- `setup.bat` 다시 실행 → Python 자동 설치
- 또는 https://www.python.org 에서 수동 설치 (PATH 체크)

**"venv 생성 실패"**
- `venv` 폴더 삭제 → `setup.bat` 다시 실행

**"패키지 설치 실패 (Visual C++ 필요)"**
- https://visualstudio.microsoft.com/visual-cpp-build-tools/ 설치

### KIS API 문제

**"토큰 발급 실패 (EGW00133)"**
- 1분에 1회 호출 제한 → 1분 후 재시도
- 또는 `data/kis_token_paper.json` 삭제하고 재시작

**"모의투자 계좌가 비어있다"**
- 한국투자 앱 → 모의투자 메뉴 → 신청 → 1억원 자동 입금

**"실거래 키로 모의투자 호출 시도"**
- KIS_PAPER 값 확인 (모의=true, 실거래=false)

### 대시보드 문제

**"http://localhost:5000 연결 안 됨"**
- 방화벽 확인 (5000 포트 허용)
- 다른 프로세스가 5000 사용 중인지 확인

**"한국 시간이 안 맞음 (장 시간 오인식)"**
- Windows 시간대 → "서울 (UTC+09:00)"로 설정

### 매매 문제

**"매수 신호인데 매수 안 됨"**
- 신뢰도 < 최소값 (모의 0.03 / 실거래 0.15)
- 매도 쿨다운 중 (1시간)
- SafetyGuard 한도 도달
- 시장 정지 발동 중
- 엄격 화이트리스트 모드 + 워치리스트 외 종목
- 적응형 임계값에서 위기장+Bear장 → 매수 차단
- 로그 또는 `[적응형]`/`[VI 차단]`/`[엄격 화이트리스트]` 메시지 확인

**"매도 신호인데 매도 안 됨"**
- 보유 0주
- 서킷브레이커 발동 (매도까지 차단)
- 같은 종목 직전에 매도함

### KIS SMS가 매일 옴
- 토큰 캐싱이 작동하지 않음
- `data/kis_token_paper.json` 파일 존재 여부 확인
- 파일이 없다면 첫 발급 시 SMS 1회, 이후 24시간마다 1회만 옴
