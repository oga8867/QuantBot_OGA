# 퀀트봇 사용법 & 코드 구조

한국/미국 주식 자동매매 봇입니다. 웹 대시보드로 조작하며, 모의투자(paper)와
실거래(KIS 한국투자증권)를 모두 지원합니다.

> ⚠️ **실거래는 실제 돈이 오갑니다.** 반드시 모의투자로 충분히 검증한 뒤,
> 소액으로 시작하세요. 모든 투자 책임은 본인에게 있습니다.

---

## 1. 빠른 시작 (새 PC에서)

1. 받은 zip을 원하는 위치에 **압축 해제**
2. **`setup.bat` 더블클릭** — 파이썬 확인 → 가상환경 생성 → 라이브러리 설치 →
   설정파일 생성 → 대시보드 자동 실행 (처음 1회, 5~10분 소요)
3. 브라우저에서 **http://localhost:5000** 자동 열림 → **설정 탭에서 API 키 입력**
4. 다음부터는 **`start.bat`** 또는 **`dashboard.bat`** 으로 대시보드 실행

> 파이썬이 없으면 setup.bat이 자동 설치를 시도합니다. 실패 시
> https://www.python.org/downloads/ 에서 3.11 버전을 "Add Python to PATH" 체크하고 설치하세요.

---

## 2. 실행 파일(.bat) 설명

| 파일 | 하는 일 | 언제 쓰나 |
|---|---|---|
| **setup.bat** | 원클릭 셋업: 파이썬 확인 → 가상환경(venv) 생성 → `requirements.txt` 설치 → `.env`/`API_KEYS.txt` 생성 → 대시보드 실행 | **새 PC 최초 1회** |
| **install.bat** | 가상환경 + 라이브러리 설치만 (대시보드 실행 안 함) | 셋업만 따로 하고 싶을 때 |
| **start.bat** | 대시보드 실행 (가상환경 필요) | **평소 대시보드 켤 때** |
| **dashboard.bat** | 대시보드 실행 (flask 없으면 자동 설치) | start.bat 대안 |
| **run_bot.bat** | 명령줄(CLI) 자동매매 모드 (`run_bot.py` 직접 실행). `--live`로 실거래 | 대시보드 없이 CLI로 돌릴 때 |
| **analyze.bat** | 명령줄 단일 종목 분석 (예: `analyze.bat 005930.KS`) | 한 종목만 빠르게 분석 |
| **packaging/pack.bat** | 다른 PC로 옮길 zip 생성 (공유용) | 봇을 다른 환경에 복제할 때 |

> 대부분은 **setup.bat → 이후 start.bat** 만 쓰면 됩니다. 나머지는 보조 도구입니다.

## 3. 코드 구조 (어느 부분이 무슨 역할인가)

```
quant-bot/
├─ run_bot.py          ★ 자동매매 봇 본체 — 분석→신호→안전장치→주문 루프 전체
├─ main.py               CLI 단일 종목 분석 진입점
├─ repair_positions.py   깨진 포지션 행 1회성 복구 도구 (필요할 때만)
├─ requirements.txt      파이썬 라이브러리 목록
│
├─ config/             ⚙️ 설정
│   ├─ settings.py        모든 설정값 정의(자본·리스크·앙상블·자본분할 등)
│   └─ user_settings.json 대시보드에서 저장한 사용자 설정 (API키 포함 — 공유 금지)
│
├─ collectors/         📥 데이터 수집
│   ├─ price_us.py        미국 주가 (yfinance)
│   ├─ price_kr.py        한국 주가 (pykrx)
│   └─ news.py            Google News RSS 뉴스 + 감성 점수 (최근 N일 필터)
│
├─ analyzers/          📊 분석
│   ├─ technical.py       기술적 지표 (RSI·MACD·볼린저·이동평균·OBV·ATR)
│   └─ news_llm.py        뉴스 LLM 요약 (로컬 GGUF/Ollama/OpenAI, 선택)
│
├─ strategy/           🧠 전략 두뇌
│   ├─ ensemble.py        앙상블 결합(기술45+팩터35+감성20) + 포지션유형 분류
│   ├─ factor.py          4대 팩터(모멘텀·밸류·퀄리티·사이즈) 점수
│   ├─ sleeves.py         전략 슬리브 신호 (자본 쿼터 모드용)
│   ├─ capital_allocator.py 전략별 자본 분할(쿼터) 관리
│   ├─ adaptive_threshold.py VIX·시장체제 기반 동적 임계값
│   └─ market_halt_detector.py 서킷브레이커/사이드카/VI 감지
│
├─ executor/           💸 주문 실행 + 청산
│   ├─ kis_executor.py    한국투자증권 실거래 API (주문·잔고·시세)
│   ├─ paper_executor.py  모의투자 체결기 (가상 자본)
│   ├─ exit_manager.py    ★ 손절·익절·트레일링·시간청산 결정 엔진
│   ├─ safety_guard.py    안전장치 (킬스위치·일일손실·종목한도)
│   └─ dual_executor.py   한국+미국 동시 운용 (선택)
│
├─ risk/               🛡️ 리스크
│   ├─ position_sizer.py  매수 수량 계산 (켈리/ATR/고정)
│   └─ stop_loss.py       ATR 기반 손절선 계산
│
├─ database/cache.py   🗄️ SQLite 저장 (거래·포지션·신호·자산기록)
│
├─ dashboard/          🖥️ 웹 대시보드 (Flask + SocketIO)
│   ├─ app.py             대시보드 서버 (상태 브로드캐스트·설정·스캐너 등)
│   ├─ templates/index.html  화면(HTML) + 가이드 탭
│   └─ static/js/main.js     화면 동작(JS)
│
├─ notifier/           📢 알림 (telegram_bot.py, discord_webhook.py)
├─ reporter/           📄 HTML 보고서 + 시장 브리핑
├─ backtest/           ⏮️ 백테스트 엔진 + 성과지표(샤프·MDD 등)
├─ scheduler/          ⏰ 작업 스케줄링
├─ utils/              🔧 공용 유틸
│   ├─ market.py          시장 판별·환율 변환
│   ├─ market_holidays.py 한국/미국 휴장일(공휴일·임시휴장) 조회
│   └─ news_filter.py     뉴스 날짜 필터 (오래된 뉴스 제외)
├─ scripts/            🧰 유틸 스크립트 (백테스트 실행 등)
└─ tests/              ✅ 자동화 테스트
```

### 매매가 일어나는 핵심 흐름 (`run_bot.py`)
1. **데이터 수집** (collectors) → 2. **분석** (analyzers + strategy) →
3. **앙상블/슬리브 신호** → 4. **포지션 사이징** (risk) →
5. **안전장치 통과** (executor/safety_guard) → 6. **주문 실행** (executor) →
7. **청산 감시** (executor/exit_manager — 매 사이클 손절/익절 체크)

## 4. 설정 & API 키

- 대시보드 **설정 탭**에서 자본·리스크·전략·API 키를 모두 조절합니다.
- KIS 실거래를 쓰려면 한국투자증권 앱에서 **Open API 신청 → 앱키/시크릿/계좌번호** 발급 후 입력.
- 저장된 설정은 `config/user_settings.json`, 키는 `.env`/`API_KEYS.txt`에 들어갑니다.
  → **이 파일들은 절대 공유/업로드하지 마세요** (.gitignore·pack 스크럽으로 보호됨).

---

## 5. 다른 PC로 공유하기

`packaging/pack.bat` 더블클릭 → 모드 선택:
- **모드 1** 코드만 (API키·기록 없음)
- **모드 2** 코드 + 거래기록 (API키 **자동 제거**) — 권장
- **모드 3** 전부 포함 (API키 평문 — 분실 주의)

생성된 `quant-bot_날짜.zip`을 새 PC에 풀고 **setup.bat** 실행하면 동일하게 동작합니다.

자세한 내용은 `packaging/패키지_사용법.md` 참고.

---

## 6. 주의사항

- 실거래 전 **모의투자(paper)** 로 최소 1~2주 검증을 권장합니다.
- 손절은 ATR(변동성) 기반이 기본이며, 고정 % 손절은 설정에서 켜야 합니다.
- KIS API는 초당 호출 한도가 있어, 잔고 조회가 가끔 지연될 수 있습니다(정상).
- 봇/대시보드는 설정 변경 후 **재시작**해야 반영됩니다.
